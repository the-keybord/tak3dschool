# Project Rules for TAK 3D School Landing Page

These are strict rules that must be followed for all modifications to this codebase. Do not bypass them without explicit user permission.

## 1. Design & Styling
- The design must be a minimalist, simple, **Material Design** aesthetic.
- The color palette is strictly limited to **orange, white, and black**.
- Avoid complex/flashy/animated components unless they align with this minimalist Material Design style.

## 2. Image Assets
- **Do NOT use any AI-generated images.**
- Use blank placeholder pictures (or simple SVG shapes/colored boxes) with clear filenames.
- The user will replace these placeholders with actual pictures and name them accordingly.

## 3. Architecture & Tech Stack
- **No backend technologies.**
- The project is strictly a static frontend landing page (HTML, CSS, and client-side JavaScript).
- It must be lightweight, simple, and runnable on any basic/cheap static hosting service (e.g., GitHub Pages, Netlify, Vercel, or simple shared hosting).
